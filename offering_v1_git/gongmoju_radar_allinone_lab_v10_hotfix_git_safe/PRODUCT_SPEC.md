# 공모주 레이더 Pro 기획 요약

## 목표

공모주 투자자가 아래 세 단계를 하나의 앱 안에서 이어서 보도록 만든다.

1. **청약 단계**: 일정, 경쟁률, 주관사, 손익분기 계산
2. **상장 단계**: 확약, 유통가능물량, 기존주주비율, 가격/기술신호
3. **이후 단계**: 보호예수 해제, 이상 변동, 전략 연계 알림

## 핵심 가치

- 흩어진 데이터를 한 화면에 모음
- 청약 정보와 상장 후 관리 포인트를 연결
- 보호예수 해제 자동매수 전략과 앱을 연결
- 운영형으로 가기 위한 캐시/업로드/스크립트 구조 확보

## 주요 데이터 소스

### 기본 내장
- sample_ipo_events.csv
- sample_market_snapshot.csv
- sample_market_history.csv
- backtest 결과 CSV

### 실데이터 / 반실데이터
- KIND 신규상장기업현황
- 38 IPO 일정 표
- DART 최근 공시
- DART 지분증권 주요정보 API
- DART 공시서류 원문 ZIP(document.xml)
- 한국투자증권 KIS 일봉/현재가
- YahooHTTP 시장 스냅샷
- 사용자가 내려받은 로컬 KIND 엑셀·CSV
- 기존 전략 프로젝트의 synthetic_ipo_events.csv

## 점수 설계

### 청약 점수
- 기관경쟁률
- 청약경쟁률 live
- 공모가 수준

### 상장 품질 점수
- 확약 비율
- 상장일 유통가능물량
- 기존주주비율
- 우리사주 실권
- 현재가 vs 공모가

### unlock pressure score
- 유통가능물량
- 기존주주비율
- 단기 가격변동
- 확약 비율 감점

## 알림 규칙

- 보호예수 해제 임박
- 이례적 가격변동
- 거래량 급증
- 과열권 / 과매도권 기술신호

## 운영 시나리오

### 수동 운영
- 사용자가 KIND 엑셀을 내려받아 앱에 업로드
- 외부 unlock dataset 경로 지정
- 앱에서 후보와 알림 확인

### 반자동 운영
- `scripts/refresh_live_cache.py`를 작업 스케줄러로 주기 실행
- `scripts/export_alerts.py`로 CSV 생성
- 이후 텔레그램/디스코드/사내 webhook 연결 가능
