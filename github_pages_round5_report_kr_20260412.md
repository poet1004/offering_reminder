# GitHub Pages 정적 웹 5차 패치

반영 내용
- 대시보드 제목을 `데일리 브리핑`으로 변경
- 브리핑 2줄 카드, `데이터 상태 보기` 링크, `곧 청약 시작`, `가까운 보호예수 해제`, `전략 요약` 카드 제거
- 요약 카드를 4개(30일 내 청약, 30일 내 상장, 환율, 시장 분위기)만 남김
- 30일 내 청약/상장 집계는 `오늘 기준 ±30일` 가까운 일정으로 계산하도록 변경
- 가까운 일정은 종목별 1줄 요약으로 묶고 보호예수 해제를 제외, 최대 8건 표시
- 공모주 달력은 날짜 칸 안에 종목명이 직접 보이도록 변경
- 청약은 시작~마감 범위를 한 줄로 표시하고 상세 패널에서도 같은 형식으로 노출
- 청약/상장/보호예수 목록 모두 페이지당 8건으로 축소
- 청약/상장 컨테이너의 정렬 기능을 구분 필터로 변경
  - 청약: 청약 예정 / 상장 후 / 전체
  - 상장: 상장 예정 / 상장 후 / 상장폐지 / 전체
- 보호예수 해제 목록은 항상 오늘과 가장 가까운 일정 순으로 정렬
- 딜 탐색기, 데이터 상태, 배포 가이드 섹션 제거
- PC 최대 폭을 줄이고 모바일 카드 2열 반응형을 강화
- CSS/JS 캐시 무효화를 위해 버전 파라미터를 `20260412r5`로 갱신

비고
- `PUBLIC_DATA_SERVICE_KEY`에는 공공데이터포털 일반 인증키(`4344...`)를 넣으면 됩니다.
- `DART_API_KEY`에는 Open DART 키(`d602...`)를 넣으면 됩니다.
- 공공데이터 키를 넣은 뒤 `mobile-feed-pages` 워크플로를 먼저 실행하고, 이어서 `github-pages-static`을 실행해야 새 피드와 새 화면이 같이 반영됩니다.
